🏦 Bank Management System (Legacy)
A terminal-based Bank Management System built in raw C++ during my first year of computer science. This project serves as a historical baseline of my foundational programming skills, focusing on procedural logic, console-based state management, and stream validation.

📝 Note: This repository hosts the original, unrefactored codebase from my freshman year. It is preserved here to showcase my technical evolution as I prepare to build a modern, fully-engineered successor from scratch.

🛠️ Technical Overview
Language: Core C++
Paradigm: Procedural Programming
Interface: Command-Line Interface (CLI)

🚀 Key Features (First-Year Milestones)
Iterative Transaction Handling: Utilizes continuous execution loops to handle user actions without crashing.
State Mutation: Manual balance adjustment logic across separate account data streams (e.g., handling simulated transfers and balance updates).
Input Validation: Hardened std::cin streams designed to catch invalid inputs and prevent edge cases like negative balance initializations.

🧠 Reflection & Next Steps
Looking back, managing multi-account states and nested conditional logic within a single terminal application was a massive milestone that sparked my passion for software engineering.
While this codebase relies heavily on raw procedural logic, it lays the groundwork for my next steps. I am currently in the scoping phase of a brand-new, standalone successor project that will scale these foundational concepts into a complete system using modern software design, database integration, and proper architectural patterns.
